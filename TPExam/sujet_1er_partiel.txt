2)La gestion d'une base de données utilise le principe de lecteur/rédacteur. 
Sélectionnez la ou les bonnes affirmations Lecteur/rédacteur actif signifie qu'il est en train d'accéder à la base de données en lecture/écriture. 
Veuillez choisir au moins une réponse. 

a. Lorsque l'on donne la priorité aux rédacteurs, plusieurs rédacteurs peuvent accéder à la base de données en même temps. 
b. Lorsque l'on donne une priorité identique, les lecteurs restent tout de même plus prioritaire que les rédacteurs. 
c. Lorsque l'on donne la priorité aux rédacteurs. Si on a un rédacteur actif et que l'on lance un lecteur puis un rédacteur, alors le second rédacteur s'exécutera avant le lecteur. 
d. Un seul rédacteur peut accéder à la base de données à la fois. 
e. Plusieurs lecteurs peuvent accéder à la base de données en même temps sans problème 
f. Lorsque l'on donne la priorité égale. Si un lecteur est actif et que l'on lance un rédacteur puis un lecteur, alors le second lecteur s'exécutera avant le rédacteur. 
g. Lorsque l'on donne la priorité aux rédacteurs. Si un lecteur est actif et que l'on lance un rédacteur puis un lecteur, alors le second lecteur s'exécutera avant le rédacteur. 

Explications :
a.❌ Faux : Lorsque la priorité est donnée aux rédacteurs, cela signifie qu'un rédacteur peut être priorisé par rapport aux lecteurs, mais plusieurs rédacteurs ne peuvent pas écrire en même temps, sinon il y aurait un risque d’incohérence des données.
b.❌ Faux : Si on donne une priorité égale, cela signifie que les demandes sont traitées de manière équitable, mais pas forcément que les lecteurs seront toujours plus prioritaires que les rédacteurs.
c.✅ Vrai : Lorsqu'un rédacteur a la priorité, si un rédacteur est actif et qu'un lecteur arrive suivi d'un autre rédacteur, le second rédacteur est exécuté avant le lecteur.
d.✅ Vrai : Un seul rédacteur peut écrire à la fois pour éviter les incohérences dans la base de données.
e.✅ Vrai : Plusieurs lecteurs peuvent lire simultanément sans problème, car la lecture n'affecte pas l'intégrité des données.
f.✅ Vrai : Si la priorité est égale et qu'un lecteur est actif, un second lecteur pourra accéder avant un rédacteur qui attend.
g.❌ Faux : Si on donne la priorité aux rédacteurs, alors un rédacteur en attente aura la priorité sur un lecteur qui arrive après.

❌ i. Faux : array[0] et array[1] ne contiennent pas la même valeur, array[0] = 380 et array[1] = 400.

5)Le fait d'utiliser IPC_CREAT en conjonction avec IPC_EXCL permet de réinitialiser le semaphore/la section de mémoire partagée 
au moment de sa création. Veuillez choisir une réponse. 
1. VRAI 
2. FAUX ✅

FAUX

Explication :
L'utilisation de IPC_CREAT | IPC_EXCL avec shmget (pour la mémoire partagée) ou semget (pour les sémaphores) ne réinitialise pas une ressource IPC existante, mais empêche simplement sa recréation si elle existe déjà.

IPC_CREAT : Crée un nouvel objet IPC (mémoire partagée, sémaphore, etc.) s'il n'existe pas encore.
IPC_EXCL : Utilisé avec IPC_CREAT, il force l'échec si la ressource existe déjà.
Si la ressource existe déjà, shmget() ou semget() retournera une erreur (EEXIST) au lieu de la réinitialiser.

Cas pratique :
Si vous voulez forcer une réinitialisation, vous devez :

Supprimer la ressource existante avec shmctl(shmid, IPC_RMID, NULL); ou semctl(semid, 0, IPC_RMID);
Recréer la ressource proprement.

dijkstra.c :
#include <sys/sem.h>
#include <stdlib.h>
#include <stdio.h>


#include "dijkstra.h"


int sem_create(key_t cle, int initval) {
        int semid;
        union semun { int val; struct semid_ds *buf; ushort *array; } arg_ctl;


        semid = semget(cle, 1 , IPC_CREAT|IPC_EXCL|0666);
        if (semid == -1)
        {
                semid = semget(cle,1, 0666);
                if (semid == -1)
                {
                        perror("Erreur semget()");
                        exit(1);
                }
        }
        else
        {
                arg_ctl.val = initval;
                if (semctl(semid, 0, SETVAL, arg_ctl) == -1)
                {
                        perror("Erreur initialisation sémaphore");
                        exit(1);
                }
        }
        return(semid) ;
}


void P(int semid)
{
        struct sembuf sempar;
        sempar.sem_num = 0;
        sempar.sem_op = -1;
        sempar.sem_flg = 0;
        if (semop(semid, &sempar, 1) == -1)
                perror("Erreur operation P");
}


void V(int semid)
{
        struct sembuf sempar;
        sempar.sem_num = 0;
        sempar.sem_op = 1;
        sempar.sem_flg = 0;
        if (semop(semid, &sempar, 1) == -1)
                perror("Erreur opération V");
}


void sem_delete(int semid)
{
        if (semctl(semid,0,IPC_RMID,0) == -1)
                perror("Erreur dans destruction sémaphore");
}
